Introdução

Tentamos ao máximo, deixar nosso framework sem dependências de Javascript. Como você já viu, até nossos tooltips são independentes. Fizemos esse esforço para que suas aplicações web fiquem o mais leve possível.

Mas nem tudo é possível sem o uso dele. Nessa seção você vai encontrar não apenas complementos para nosso framework como também snippets úteis no dia-a-dia. E sabe porque? Porque não aguentamos mais reescrever códigos. E achamos que você também. Por isso, estamos adotando o conceito DRY aqui. Tudo o que nos é útil será disponibilizado para vocês também.

Além disso, usamos vários scripts de terceiros e mostramos como fizemos a integração deles com nosso framework.

Basta inserir o script na página que suas tabs passarão a funcionar automaticamente. Para isso, basta seguir a estrutura do HTML abaixo:

<div class="tabs">
    <button href="#tb1" type="button" class="ON">Botão 1</button>
    <button href="#tb2" type="button">Botão 2</button>
</div>
<div class="col tab-content">
    <div id="tb1" class="tab-pane ON">Conteúdo 1...</div>
    <div id="tb2" class="tab-pane">Conteúdo 2...</div>
</div>

Para os botões, basta seguir o que explicamos em Navegação. Da mesma forma, o conteúdo precisa ser envolvido por um elemento com class="tab-content" e os elementos que irão aparecer e desaparecer, devem possuir class="tab-pane". Além disso, use um link de âncora no tab, pra chamar o class="tab-pane" com o ID respectivo. Defina o item que estará ativo no início utilizando class="ON" tanto no botão quanto no conteúdo.

Lazy

<script src="assets/script/libs/horseggs/egg.lazy.min.js"></script>

Alguns eventos funcionariam melhor se fossem disparados apenas quando o usuário finaliza o evento, não acha? Vamos tomar por exemplo o resize.

  • No Firefox o evento é disparado apenas no final do redimensionamento (como sugeri acima);
  • Mas no IE, Chrome e Safari, o disparo é realizado freneticamente durante todo redimensionamento;
  • E no Opera, todos os eventos são gravados e disparados apenas no final.

Não seria bom se pudéssemos controlar isso? É pra isso que usamos o Lazy. O Lazy deixa seu evento com preguiça, fazendo-o disparar apenas uma vez, antes ou depois do evento, depois de um período de tempo específico. Vamos lá, redimensione seu browser e veja abaixo a diferença:

Eventos registrados: 0
Eventos registrados: 0

Com certeza você já deve estar imaginando diversas utilidades para este script. Aliás, compartilhe suas ideias conosco nos comentários, ok? Mas três são bem óbvias: resize (apresentada acima), scroll e keyup. Você pode, por exemplo, mover um elemento após o scroll ou apresentar sugestões num campo de busca apenas quando o usuário parar de digitar. Veja abaixo:

  • Sugestão 2

Como funciona?

Veja no markup abaixo como botar pra rodar:

<script type="text/javascript">
$.egg.lazy('resize');
$(window).resize( function () { ... } );
</script>

A única coisa que precisamos fazer é ativar as modificações no evento nativo do jQuery. Para isso, basta passar o nome do evento para nosso script (linha 2). Depois disso, é só usar o evento como normalmente o faria (linha 3).

Isolando a atuação do Lazy

Se você não quiser alterar a função nativa do jQuery, também não tem problema. Você também pode aplicar o Lazy direto na função de callback. Dessa forma, o Lazy será aplicado apenas nesta função, não interferindo nas demais.

<script type="text/javascript">
var lazyFunction = $.egg.lazy(callback);
$(window).resize(lazyFunction);
</script>

Personalização

Além disso, você tem algumas opções à sua disposição:

<script type="text/javascript">
$.egg.lazy('resize', { delay: 200, immediate: false });
// OU
var lazyFunction = $.egg.lazy(callback, { delay: 200, immediate: false });
</script>

O delay altera o tempo que o script irá esperar pra chamar sua função após o evento acabar. Já immediate pode fazer sua função ser chamada logo no início do evento (ao invés do final). Basta marcar immediate: true.

Scroll 2

<script src="assets/script/libs/horseggs/egg.scroll2.min.js"></script>

Com essa simples função você pode deslizar o scroll do usuário automaticamente até uma determinada posição, como o topo da tela ou âncoras, espalhadas pelo conteúdo.

Deslizando até um elemento

<script type="text/javascript">
    $.egg.scroll2('#scroll-2-top');
</script>

Basta passar o ID do elemento desejado para levar seu usuário até ele.

Voltar ao topo

Agora, para voltar ao topo, basta indicar 'top'.

<script type="text/javascript">
    $.egg.scroll2('top');
</script>

Adicionando um recuo

Ás vezes algum elemento do DOM esteja interferindo no scroll, e ele esteja um pouco fora do ideal. Para compensar isso, você pode passar um valor de recuo (em pixels) pra função. Dessa forma você elimina a altura de qualquer elemento do seu layout que possa estar "confundindo" o scroll.

<script type="text/javascript">
    $.egg.scroll2('#topic', 192);
</script>

Scrolled

<script src="assets/script/libs/horseggs/egg.scrolled.min.js"></script>

Mais uma função envolvendo o scroll. Essa aqui serve para destacar itens do seu menu de acordo com a posição do scroll. Nosso menu lateral, por exemplo, está fazendo uso desta função. Legal, não? Basta deslizar a página para notar seu funcionamento. A medida que o scroll atinge uma seção, seu respectivo link no menu é destacado.

Como funciona?

Basta inserir class="scrolled" no menu que você deseja que receba a ação. Nosso script irá buscar automaticamente todos os elementos que possuam o ID especificado nos links desse menu.

<a href="#content-1">Link 1</a>
<a href="#content-2">Link 1</a>
<div id="content-1">Conteúdo 1</div>
<div id="content-2">Conteúdo 2</div>

Em seguida, nosso script irá calcular automaticamente a distância entre os elementos e acrescentar a class="ON" nos links do menu, quando o scroll os atingir. Aí cabe a você definir como ele será destacado.

Atualizações no DOM

Caso algum outro script faça modificações no DOM, atente-se de que isso irá modificar a posição dos elementos. Por isso, é necessário incluir o código abaixo dentro do seu script, para atualizar as posições após a alteração do DOM.

<script type="text/javascript">
    $.egg.scrolled.refresh();
</script>

Análise de Eventos do You Tube

<script src="assets/script/libs/horseggs/egg.youtube.tracker.min.js"></script>

Gostaria de visualizar informações dos seus vídeos no Google Analytics? Por exemplo, se estão assistindo até o fim ou quantos estão clicando para assistir? Então você encontrou a solução!

Como funciona?

Insira nosso script na sua página e em seguida, inicie o script da seguinte forma:

<script type="text/javascript"> $.egg.youtube.tracker.start(); </script>

Nosso script irá rastrear todos os iframes do You Tube presentes na sua página, e em seguida, como num passe de mágica, ele irá registrar os eventos do seu vídeo.

Assim você poderá acompanhar o impacto dessa interação no seu site.

Personalizando os Eventos

Você também tem a possibilidade de personalizar os termos que aparecem em seu Google Analytics. Veja como:

<iframe src="http://www.youtube.com/embed/" title="PSY - Gangnam Style" data-category="Vídeo na Página de Produto" frameborder="0"></iframe>

Na tag <iframe> você pode alterar o nome do vídeo que aparecerá no You Tube atribuindo um valor a title, como feito acima. Se você não adicionar um title, usaremos o ID do vídeo no YouTube para diferenciar o conteúdo. Além disso, você pode definir uma nova categoria para o evento - caso não deseje usar o valor padrão (Videos) - usando data-category. Você também pode personalizar os termos que indicam quando o vídeo foi exibido, pausado e concluído acrescentando os atributos data-play-state, data-pause-state e data-end-state.

<script type="text/javascript">
$.egg.youtube.tracker.start({ category: 'Vídeo', status: { play: 'Exibindo', pause: 'Pausado', end: 'Concluído' }});
</script>

Se preferir, você pode personalizar os termos de todos os vídeos da página de uma só vez na própria função, como fizemos acima. Veja:

Código Termo
{ category: ''} O termo que deve aparecer para marcar a categoria raiz desse evento.
{ status: { play: '' }} O termo que deve aparecer para marcar que o vídeo foi iniciado.
{ status: { pause: '' }} O termo que deve aparecer para marcar que o vídeo foi pausado.
{ status: { end: '' }} O termo que deve aparecer para marcar que o vídeo foi assistido até o fim.

Atualizações no DOM

Caso seu projeto insira vídeos na página após o primeira leva, você terá que registrar a presença destes novos vídeos no nosso script. Para isso, basta você usar a função abaixo. Nosso script é inteligente o suficiente para realizar um novo rastreamento, eliminando os vídeos já analizados.

<script type="text/javascript">
    $.egg.youtube.tracker.refresh();
</script>

Para mais informações sobre o rastreamento de eventos, consulte a documentação do Google Analytics.
