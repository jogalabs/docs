    Lazy

    <script src="assets/script/libs/horseggs/egg.lazy.min.js"></script>

    Alguns eventos funcionariam melhor se fossem disparados apenas quando o usuário finaliza o evento, não acha? Vamos tomar por exemplo o resize.

    • No Firefox o evento é disparado apenas no final do redimensionamento (como sugeri acima);
    • Mas no IE, Chrome e Safari, o disparo é realizado freneticamente durante todo redimensionamento;
    • E no Opera, todos os eventos são gravados e disparados apenas no final.

    Não seria bom se pudéssemos controlar isso? É pra isso que usamos o Lazy. O Lazy deixa seu evento com preguiça, fazendo-o disparar apenas uma vez, antes ou depois do evento, depois de um período de tempo específico. Vamos lá, redimensione seu browser e veja abaixo a diferença:

    Eventos registrados: 0
    Eventos registrados: 0

    Veja no markup abaixo como botar pra rodar:

    <script type="text/javascript">
$.lazy('resize');
$(window).resize( function () { ... } );
</script>

    A única coisa que precisamos fazer é ativar as modificações no evento nativo do jQuery. Para isso, basta repetir o que foi feito na linha 2, passando o nome do evento. Depois é só usar o evento como normalmente o faria (linha 3).

    Com certeza você já deve estar imaginando diversas utilidades para este script. Aliás, compartilhe suas ideias conosco nos comentários, ok? Mas três são bem óbvias: resize (apresentada acima), scroll e keyup.     Se você não quiser alterar a função nativa do jQuery, também não tem problema. Você também pode aplicar o Lazy direto na função de callback. Dessa forma, o Lazy será aplicado apenas nesta função, não interferindo nas demais.

    <script type="text/javascript">
var lazyFunction = $.lazy(callback);
$(window).resize(lazyFunction);
</script>

    Além disso, você tem algumas opções à sua disposição:

    <script type="text/javascript">
$.lazy('resize', { delay: 200, immediate: false });
// OU
var lazyFunction = $.lazy(callback, { delay: 200, immediate: false });
</script>

    O delay altera o tempo que o script irá esperar pra chamar sua função após o evento acabar. Já immediate pode fazer sua função ser chamada logo no início do evento (ao invés do final). Basta marcar immediate: true.
