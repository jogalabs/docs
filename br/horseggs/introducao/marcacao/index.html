    Marcação

    Se você já baixou uma cópia do nosso framework, vai notar que na raiz existe um arquivo chamado template.html. Nele você encontrará o seguinte markup:

    <!DOCTYPE html>
<!--[if IE 8]>        <html class="lt-ie9"> <![endif]-->
<!--[if gt IE 8]><!--><html> <!--<![endif]-->
<head>

<meta charset="utf-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1" />
<meta name="viewport" content="width=device-width, initial-scale=1" />
<title>Template</title>
<link rel="stylesheet" type="text/css" href="assets/style/horseggs.css" />
<link rel="stylesheet" type="text/css" href="assets/style/start.css" />

</head>
<body>

<h1>Olá Mundo!</h1>

<!--[if lt IE 9]>
<script src="assets/script/ie.polyfills.js"></script>
<![endif]-->
<script src="assets/script/includes.min.js"></script>
<script src="assets/script/start.min.js"></script>

</body>
</html>

    Seguem alguns pontos sobre o markup acima:

    Doctype

    Começamos usando o DOCTYPE padrão do HTML5. Essa tag diz para o browser o que ele vai encontrar aqui e como renderizar a página. Não se preocupe: irá funcionar até em browsers antigos como o Internet Explorer 8.

    Meta Tags

    Definimos a codificação da página para UFT-8, que é a universal. Em seguida, garantimos que o Internet Explorer use o melhor modo de renderização possível, e se tiver instalado, o Chrome Frame. Por fim, pedimos para que o browser use a largura do dispositivo mobile, ao invés de tentar simular um desktop escalando o conteúdo. Sem isso, nossas media queries não seriam aplicadas.

    Condicionais

    Usamos as condicionais do Internet Explorer para acrescentar uma classe ao elemento <html> para poder, mais tarde, aplicar estilos específicos para estes browsers (se necessário).

    Folhas de Estilo

    Nossa experiência mostrou ser uma boa prática trabalhar com dois arquivos separados: um apenas com os estilos do Horseggs e outro com o layout da sua página. Dessa forma, o arquivo CSS do Horseggs, que é um pouco mais pesado, virá do cache, já que ele será chamado em todas as páginas do seu site. Em outra requisição você terá um arquivo muito menor, apenas com o estilo daquela página. A mesma regra se aplica ao javascript, que você verá em seguida.

    Javascript

    Primeiro, incluímos o Compilamos todos os javascripts do Horseggs (que são bem leves) em um único arquivo, para que você faça uma única requisição. Em seguida, incluímos outro arquivo, em branco, que será usado para sua própria programação. Para o IE (sempre ele) criamos um arquivo chamado "ie.polyfills.js" que compila alguns, como HTML 5 Shiv e o REM. Este, é chamado apenas através de uma condicional.

    É uma boa prática deixar os javascripts para o final, pois dessa forma seu usuário poderá visualizar todo conteúdo da página, mesmo que ela apresente erros. Além disso, sempre que possível faça uso de redes de distribuição de conteúdo (CDNs) para chamar scripts famosos como o jQuery. As vantagens são várias. Acompanhe:

    • Existe uma grande possibilidade do seu usuário já ter acessado outra página que possua o script que você necessita. Se esse for o caso, o browser irá detectar que o arquivo já está na máquina do seu usuário, eliminando a necessidade de um novo download - o que acelera a entrega do seu conteúdo.
    • Estas empresas geralmente possuem seu conteúdo distribuído em diversos servidores por todo o mundo. Dessa forma, caso ele precise baixar o arquivo, poderá pegá-lo mais próximo, fisicamente, de onde ele está. E o melhor: você não pagou nada a mais por isso.
    • Se ele for baixar o arquivo direto da CDN, tenha certeza de que a estrutura deles é absurdamente melhor que a sua, não importa quantos dólares você gaste por mês nisso.
    • Além disso, a maioria dos browsers geralmente limitam o número de downloads simultâneos de um mesmo domínio em até 4. Se você buscar esse arquivo de uma CDN, você está furando a fila. ;)
