    Introdução

    Formulários são chatos. É isso, sem meios termos. É um fato. Mas na maioria das vezes, todo aquele layout incrível só existe por causa dele. No final das contas seu objetivo é conseguir um lead ou uma compra, e o usuário vai acabar se deparando com um destes.

    Mas seus dias de terror estão pertos do fim. Com nosso framework você pode montar de simples fomulários de login a extensos e complexos formulários de compra. Tudo isso sem criar uma única classe de CSS. Pode ir pra cima que a gente te dá cobertura. ;)

    Uma classe faz toda diferença

    Preparamos uma formatação especial para automaticamente deixar seus formulários bonitos e padronizados. Para isso você precisa de apenas uma coisa: class="standard". Basta inserir essa classe nos elementos <legend>, <input>, <textarea>, <input type"radio"> e <input type"checkbox"> para deixá-los com outra cara.

    <div class="row">
    <div class="col-4x"><label>Input Label</label><input type="text" /></div>
    <div class="col-4x"><label>Input Label</label><input type="text" class="standard" /></div>
</div>

    Estrutura

    Como você percebeu acima, este elemento com class="standard" ocupa automaticamente 100% da largura do elemento que o contém. Dessa forma, toda estrutura do formulário é montada como peças de Lego, usando as linhas e colunas do nosso sistema de grid responsivo, ou seja, da mesma forma que você já vem fazendo com o seu layout.

    <div class="row">
    <div class="col"><legend class="standard">Legend</legend></div>
</div>
<div class="row">
    <div class="col-4x"><label>Input Label</label><input type="text" class="standard" /></div>
    <div class="col-4x"><label>Input Label</label><input type="text" class="standard" /></div>
    <div class="col-4x"><label>Input Label</label><input type="text" class="standard" /></div>
</div>
    Legend

    Se você quiser que a label esteja na mesma linha do campo de texto, não tem problema: Basta separar cada um em uma coluna diferente.

    <div class="row">
    <div class="col span-3-8"><label>Input Label</label></div>
    <div class="col-4x"><input type="text" class="standard" /></div>
    <div class="col span-3-8"><label>Input Label</label></div>
    <div class="col-4x"><input type="text" class="standard" /></div>
</div>

    Botões de seleção

    É sempre bom ter opções. Disponibilizamos checkboxes e botões de radio personalizados com estilo clean para selecionar uma ou mais opções em uma lista. Magia? Não, apenas CSS.

    <input type="checkbox" class="standard" id="sample-1" />
<label for="sample-1">Exemplo 1</div>
...
<input type="radio" class="standard" id="sample-2" />
<label for="sample-2">Exemplo 1</div>

    Além de adicionar class="standard", você também precisa seguir o markup acima, aplicando a <label> após o <input>.

    Você pode personalizar o ícone que aparece dentro dos checkboxes. Para isso, basta adicionar uma das classes abaixo ao <input="checkbox">.

    • class="plus"
    • class="minus"
    • class="cross"
    • class="asterisk"

    Por padrão, o switch fica sempre à direita. Mas você pode mudar isso acrescentando class="switch left".

    Campos de seleção

    Estes campos, por padrão, não podem ser totalmente estilizados. Mas conseguimos fazer com que ele fique no mesmo padrão estético do restante do formulário. Para isso, você precisa envolver seu select field com uma div específica, conforme o exemplo abaixo:

    <div class="standard select">
    <select name="select">
        <option value="1">1</option>
        <option value="2">2</option>
        <option value="3">3</option>
        <option value="4">4</option>
        <option value="5">5</option>
    </select>
</div>

    Campos de texto

    Campos que funcionam de forma similar ao input, mas com opção de múltiplas linhas. Controle o tamanho usando a propriedade nativa rows="".

    <textarea class="standard" name="textarea" rows="3"></legend></div>

    Prefixo & Sufixo

    Adicione texto antes ou depois dos campos de texto do seu formulário. Veja como pode ser útil:

    @
    http:// .seu-site.com

    Para conseguir este resultado, basta seguir a markup abaixo. Não se esqueça de envolver os elementos com <div class="input collapse">, ok?

    <div class="input collapse">
    <span class="prefix">http://</span>
    <input type="text" class="standard" placeholder="subdomínio">
    <span class="suffix">.seu-site.com</span>
</div>

    Validação

    Temos alguns estilos pré-definidos para passar mensagens de erro, alerta ou sucesso. Ative esse estilo aplicando class="error", class="warning" ou class="success" ao seu <input>.

    Campos desabilitados

    Adicione a propriedade disabled ou readonly ao seu input para prevenir a entrada de dados pelo usuário e mudar ligeiramente o visual. Especialmente útil para evitar que seu usuário clique duas vezes no botão Submit, gerando duplicidade no seu banco de dados.

    <input type="text" disabled />
<input type="text" readonly />
<div class="standard select disabled">
<select name="select" disabled>...</select>
</div>
<textarea name="textarea" rows="2" disabled>Mensagem desabilitada...</textarea>
    @

    Tamanho do formulário

    Nosso formulário possui campos num tamanho agradável para leitura, pensado para dispositivos de toque. Mas caso tenha pouca área para apresentar seu formulário, você pode modificar a altura de todos os campos adicionando apenas uma classe à tag Fieldset. Use "moderate" para um formulário ligeiramente menor e "small" para um formulário realmente pequeno.

    <fieldset class="moderate">...</fieldset>
    @ 
    <fieldset class="small">...</fieldset>
    @

    É isso aí: com uma única classe, você reformatou todo o formulário sem esforço.

    Tamanho dos campos

    Por padrão, todos os campos do tipo texto, assim como áreas de texto, possuem a mesma medida da coluna que os contém. Porém, é uma boa prática definir medidas para cada campo que se aproximem da resposta que você espera do usuário. Dessa forma, ele se sentirá mais confiante de que o que está digitando está correto. Para agilizar seu desenvolvimento, criamos uma série de medidas pré-definidas.

    Funciona da seguinte forma: Quantos caracteres você espera que o usuário digite nesse campo de "nome"? 15 caracteres é um bom número, não acha? Então acrescente class="maxlength-15" a esse campo. Fácil, não? Funciona assim, praticamente da mesma forma que o nosso sistema de grid. Essa função é ainda mais útil quando tratamos de numerais. Por exemplo, num campo de CPF, usaríamos class="maxlength-11", e num campo de valor unitário, class="maxlength-1".

    <div class="row">
<div class="col span-2"><label>Name</label></div>
<div class="col span-6"><input type="text" class="standard maxlength-15" /></div>
<div class="col span-2"><label>CPF</label></div>
<div class="col span-6"><input type="text" class="standard maxlength-11" /></div>
<div class="col span-2"><label>Quantidade</label></div>
<div class="col span-6"><input type="text" class="standard maxlength-1" /></div>
</div>
