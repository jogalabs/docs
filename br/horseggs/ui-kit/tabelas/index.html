    Tabelas

    A web como a conhecemos hoje deve muito às suas linhas e colunas. Quem é das antigas, sabe. E ainda hoje, elas ainda são usadas - por exemplo, no markup de e-mail marketing. Mas elas não foram esquecidas. Só voltaram a exercer o papel para o qual foram criadas: exibir dados.

    A Estrutura

    Tabelas são organizadas por linhas (<tr>) e colunas (<td>). Além disso, existe uma marcação especial para a primeira linha, que geralmente define o tipo de dado que cada coluna terá. No <head>, a coluna muda de <td> para <th>.

    <table class="standard">
    <thead>
        <tr>
            <th>Table Header</th>
            <th>Table Header</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Content Goes Here</td>
            <td>This is longer content Donec id elit non mi porta gravida at eget metus.</td>
        </tr>
        <tr>
            <td>Content Goes Here</td>
            <td>This is longer Content Goes Here Donec id elit non mi porta gravida at eget metus.</td>
        </tr>
        <tr>
            <td>Content Goes Here</td>
            <td>This is longer Content Goes Here Donec id elit non mi porta gravida at eget metus.</td>
        </tr>
    </tbody>
</table>

    Para não interferir em outras tabelas de terceiros, você ganha nosso estilo ao aplicar class="standard". A partir desse momento, sua tabela começará a ser bem tratada.

    Tabelas Responsivas

    Todas as tabelas acima funcionam perfeitamente em qualquer dispositivo, já que nosso framework foi projetado desde o início para ser responsivo. Mas em alguns casos, pode ser interessante definir uma largura mínima para sua tabela, para que a leitura dos dados não seja prejudicada.

    Para isso, envolva sua tabela com <div class="table-min-width">. Por padrão, a largura mínima da tabela será de 768 pixels, mas você pode sobreescrever esse valor em seu arquivo CSS.

    Vamos lá: teste em seu smartphone e veja que nosso framework cria um scroll apenas na tabela.

    <div class="table-min-width">
    <table class="standard">...</table>
</div>

    O ideal, entretanto, seria criar uma segunda tabela, apenas com as colunas mais importantes, e depois alternar entre os dispositivos usando class="visible mobile" na tabela otimizada e class="hidden mobile" na tabela padrão. Ficaria assim:

    <table class="standard visible mobile">
    // PARA SMARTPHONES
</table>
<table class="standard hidden mobile">
    // PARA OS DEMAIS APARELHOS
</table>

    E que tal pedir para seu usuário posicionar o celular no modo paisagem (deitado) para visualizar a tabela? Para isso, acrescente class="landscape-please" na sua tabela. Em seguida crie um <span class="landscape-message"> com uma mensagem pedindo para o usuário girar o aparelho. Confira no seu celular.

    <table class="standard landscape-please">...</table>
<span class="landscape-message">Por favor, gire seu aparelho para visualizar melhor a tabela acima.</span>
